   - Run `npx prisma db push`

2. Create src/lib/reminders/reminder-email.ts:
   - Import "server-only"
   - sendDocumentReminderEmail function accepting:
     { clientEmail: string, clientName: string | null, missingDocuments: string[], advisorName: string, advisorFirmName: string, advisorLogoUrl?: string, portalUrl: string }
   - Uses Resend (same pattern as src/lib/email.ts and src/lib/invitations/email.ts)
   - Renders branded HTML email template:
     - Advisor logo (if HTTPS URL exists) at top
     - Greeting with client name
     - "You have X documents still needed" message
     - Bulleted list of missing document names
     - CTA button linking to /documents portal
     - Advisor signature block with firm name
   - Returns SendEmailResult type (same as invitations/email.ts)

3. Create src/lib/reminders/document-reminders.ts:
   - Import "server-only"
   - processDocumentReminders function:
     a. Query all unfulfilled DocumentRequirements where:
        - fulfilled = false
        - createdAt is more than 3 days ago (grace period before first reminder)
        - lastReminderSentAt is null OR more than 7 days ago (deduplication)
     b. Group by clientId to send one email per client with all missing docs
     c. For each client group:
        - Load client email/name from User
        - Load advisor info (name, firmName, logoUrl) from AdvisorProfile
        - Call sendDocumentReminderEmail
        - Update lastReminderSentAt on all included requirements
     d. Return { clientsReminded: number, documentsIncluded: number }
   - Use proper error handling: if one client fails, log and continue to next

4. Create src/app/api/cron/document-reminders/route.ts:
   - GET handler (cron endpoints typically use GET)
   - Protect with a CRON_SECRET env var check: request must include Authorization: Bearer ${CRON_SECRET} header. If missing or wrong, return 401.
   - Call processDocumentReminders()
   - Return JSON with results: { clientsReminded, documentsIncluded }
   - This endpoint can be triggered by Vercel Cron, external cron service, or manual curl

Note: Do NOT use node-cron (in-process scheduler). Use an HTTP endpoint that external cron services call. node-cron does not survive serverless function cold starts. The research recommended node-cron but this is a serverless Next.js app — use Vercel Cron or external scheduler pattern instead.
