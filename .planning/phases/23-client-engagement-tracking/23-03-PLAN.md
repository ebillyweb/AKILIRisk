    1. Export type EngagementMetrics with fields:
       - overallCompletionPct: number (0-100)
       - activeClientCount: number
       - stalledClientCount: number
       - overdueMilestoneCount: number

    2. Export type EngagementClientRow with fields:
       - clientId: string, clientName: string, clientEmail: string
       - completionPct: number, completedCount: number, totalCount: number
       - lastActivityAt: Date | null
       - blockedCount: number
       - isStalled: boolean

    3. Export async function getEngagementMetrics(advisorProfileId: string): Promise of EngagementMetrics
       - Query all clients assigned to this advisor via clientAdvisorAssignment (status ACTIVE)
       - For each client: find assessments with actionPlanPublishedAt not null
       - Count milestones by status across all published assessments
       - Overall completion: (completed milestones) / (total milestones) * 100
       - Active clients: count of clients with at least one published action plan
       - Stalled: clients where the most recent SolutionActivity createdAt is older than subDays(new Date(), 14) and they have IN_PROGRESS recommendations
       - Overdue milestones: count of milestones with dueDate before now() and status NOT in terminal (COMPLETED, SKIPPED, DEFERRED)

    4. Export async function getEngagementClients(advisorProfileId: string, filter?: "all" | "stalled"): Promise of EngagementClientRow[]
       - Returns per-client engagement data for the dashboard table
       - When filter is "stalled": only return clients where isStalled is true

    5. Export async function getUpcomingMilestones(advisorProfileId: string, daysAhead?: number): Promise of array
       - Returns milestones with dueDate within next N days (default 30)
       - Grouped by client, include milestone title, recommendation name, dueDate, status

    6. Export async function getPortfolioEngagementData(advisorProfileId: string): Promise of Map of clientId to { completedCount, totalCount, blockedCount }
       - Lightweight query for the portfolio engagement column (D-07)
       - Only returns data for clients with published action plans
       - Clients without published plans are not in the map (UI renders "--")
