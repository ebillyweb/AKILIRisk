## Layout Patterns

### Activity Feed (inline on Strategic Action Plan page)

- Renders as `Collapsible` section below existing content on the Strategic Action Plan page (D-06)
- Trigger: "Recent Activity ({count})" — only renders when `activities.length > 0` (D-06 zero-state: section does not exist)
- Content: reverse-chronological list grouped by date
- Date group header: `text-xs text-muted-foreground font-semibold` on `bg-muted/50` strip, sticky within scroll
- Each activity row: 24px-wide dot column (left) + content column (right)
- Dot column: 8px colored circle with 2px vertical connector line (`bg-border`)
- Content column: action description (`text-sm`), recommendation name (`text-xs text-muted-foreground`), relative timestamp (`text-xs text-muted-foreground`, right-aligned)
- Pagination: "Show more" `Button variant="ghost"` at bottom, loads 20 items per page

### Next Step Callout (on client Action Plan page)

- Positioned above the "Immediate Priorities" section (D-09)
- `Card` with 3px left border using `--brand` color
- Interior: icon (`AlertCircle` from lucide, 20px) + heading "Next Step" (`text-sm font-semibold`) + milestone title (`text-sm`) + due date badge (`Badge variant="outline"`, shows date or "No due date")
- If no incomplete milestones with due dates exist, falls back to the first incomplete milestone by sort order
- Does not render if all milestones are terminal

### Milestone Progress Bars (extending ProgressDashboard)

- Each recommendation card gains a `Progress` bar below the existing task status selector
- Label: "{completed}/{total} milestones" (`text-xs text-muted-foreground`)
- `Progress` bar: fill uses `--brand` / `--trust-accent`, height 6px (`h-1.5`)
- Read-only milestone checklist below progress bar: `Collapsible` with list of milestone titles + `MilestoneStatusBadge` per row
- Client cannot interact with milestones (D-10 read-only)

### Engagement Column (advisor portfolio table)

- New column in existing `RecommendationsPortfolio` TanStack React Table
- Column header: "Engagement"
- Cell content for published action plans: "{completedCount}/{totalCount}" + circular completion percentage badge
- Cell content for unpublished/no action plan: "--" (em dash, `text-muted-foreground`) (D-07 zero noise)
- BLOCKED indicator: small red dot (6px) next to percentage when any milestone is BLOCKED (D-12)
- Column is conditionally rendered: only appears when enterprise feature flag `implementationTrackingEnabled` is true (D-01)

### Engagement Dashboard Page (`/advisor/engagement`)

- Full-width `page-shell` layout
- `AdvisorPageHeader` with kicker "Engagement", title "Implementation Tracking", subtitle "Track client progress across published action plans"
- Top row: 4 metric cards in responsive grid (`grid-cols-1 md:grid-cols-2 xl:grid-cols-4`, gap `xl` 32px)
  - Card 1: "Overall Completion" — percentage + `Progress` bar
  - Card 2: "Active Clients" — count of clients with published action plans
  - Card 3: "Stalled" — count of clients with no milestone activity in 14 days (warning color)
  - Card 4: "Overdue Milestones" — count of milestones past due date (destructive color if > 0)
- Below metrics: `Tabs` with "All Clients" | "Stalled" | "Upcoming Milestones"
  - All Clients tab: `Table` with columns: Client Name, Completion %, Last Activity, Blocked Count, Actions
  - Stalled tab: filtered view of clients with no activity in 14 days
  - Upcoming Milestones tab: list of milestones with due dates in next 30 days, grouped by client
- Page only accessible when enterprise feature flag is enabled; returns redirect otherwise

### Publish Action Plan (advisor action)

- "Publish Action Plan" `Button variant="default"` (primary accent) on advisor guidance page
- Position: top-right of the guidance page header, next to existing controls
- Click: confirmation `Dialog` — "Publish this action plan? [Client Name] will be able to see their Strategic Action Plan and track progress."
- Confirm: "Publish" (accent). Cancel: "Not yet" (ghost).
- After publish: button changes to `Badge variant="success"` reading "Published {date}" (non-interactive)

### Block Milestone Dialog (advisor action)

- Triggered from milestone status dropdown when selecting "Blocked"
- `Dialog` with required reason field (`Textarea`, min 10 characters)
- Confirm: "Block Milestone" (destructive variant). Cancel: "Cancel".

### Defer Milestone Dialog (advisor action)

- Triggered from milestone status dropdown when selecting "Deferred"
- `Dialog` with required reason (`Textarea`) + optional revisit date (`Input type="date"`)
- Follows same pattern as Phase 22 Defer Dialog
- Confirm: "Defer Milestone" (outline variant). Cancel: "Cancel".

## State Definitions

Every new component must handle all applicable states:

| State | Treatment |
|-------|-----------|
| Loading | `Skeleton` placeholders matching component dimensions |
| Zero-state | Component returns `null` — no empty placeholders (D-06 principle) |
| Error | `Alert` variant="destructive" with problem + retry action |
| Loaded | Standard rendered content |

Exceptions to the "return null" zero-state rule:
- Engagement Dashboard page: shows "No published action plans" empty state because the user navigated explicitly (not inline)
- Engagement column in portfolio: shows "--" for no data (not null, the column itself always renders when flag is on)

## Interaction Contracts

### Activity Feed (D-04, D-05, D-06)

- Collapsible default state: collapsed. User clicks trigger to expand.
- Role filtering: server-side. Advisor sees all activity types. Client sees own task status changes + advisor-visible notes only. No client-side filtering toggle.
- Date grouping: "Today", "Yesterday", then "MMM d, yyyy" for older dates (date-fns `format`).
- Pagination: initial load 20 items. "Show more" appends 20 more. No infinite scroll.

### Milestone Status Changes (advisor only, D-10, D-11)

- Advisor changes milestone status via `Select` dropdown on each milestone row
- NOT_STARTED -> IN_PROGRESS -> COMPLETED: immediate optimistic UI, no dialog
- Any status -> BLOCKED: opens Block Milestone Dialog (reason required)
- Any status -> DEFERRED: opens Defer Milestone Dialog (reason required)
- Any status -> SKIPPED: immediate, no dialog
- Status change logs `SolutionActivity` automatically (server-side)

### Auto-Completion (D-12)

- When all milestones reach terminal state (COMPLETED, SKIPPED, DEFERRED): recommendation badge auto-transitions to "Completed"
- UI: after milestone status save, if auto-completion triggers, show inline toast/banner: "All milestones complete -- [Recommendation Name] marked as completed"
- Toast uses `Alert` variant="default" with success icon, auto-dismisses after 5 seconds
- Any BLOCKED milestone: recommendation stays IN_PROGRESS, BLOCKED dot appears in portfolio engagement column

### Publish Action Plan (D-02, D-03)

- Advisor clicks "Publish Action Plan" -> confirmation Dialog -> sets `publishedAt` timestamp
- After publish: all tracking surfaces activate for this client
- Reversibility: no unpublish in this phase (deferred). Button becomes read-only badge after publish.
- Zero noise: if advisor never clicks Publish, client action plan page still works as Phase 22 designed it — no tracking surfaces appear

### Feature Flag Gating (D-01)

- All engagement surfaces check `implementationTrackingEnabled` server-side before rendering
- If flag is off: engagement column hidden, engagement dashboard page redirects, activity feed section never renders, milestone progress bars hidden, Next Step callout hidden
- Publish button still appears (it works independently of tracking flag — publishing the action plan is always available)

## Accessibility Requirements

- Activity feed timeline uses `role="feed"` with `aria-label="Recent activity"`. Each item uses `role="article"`.
- Milestone status `Select` dropdowns must announce status change via `aria-live="polite"` on the parent milestone row.
- Milestone progress bars use `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="{Recommendation Name} milestone progress"`.
- Color is never the sole differentiator for milestone status -- all badges pair color with text label.
- BLOCKED dot indicator in portfolio engagement column includes `aria-label` with blocked count.
- Next Step callout card uses `role="status"` with `aria-live="polite"`.
- Engagement dashboard metric cards include `aria-label` describing the metric value.
