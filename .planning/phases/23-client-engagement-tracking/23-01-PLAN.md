    1. Add three new action constants to SOLUTION_ACTIONS:
       - AUTO_COMPLETED: "auto_completed"
       - MILESTONE_BLOCKED: "milestone_blocked"
       - MILESTONE_DEFERRED: "milestone_deferred"

    2. Extend updateMilestoneStatus input type to accept:
       - status union: add "BLOCKED" | "DEFERRED"
       - reason?: string (required when status is BLOCKED, optional for DEFERRED)
       - revisitDate?: Date (optional, for DEFERRED only)

    3. In the transaction body of updateMilestoneStatus:
       - When status is BLOCKED: set blockedReason from reason, clear deferredReason/deferredRevisitDate, set completedAt to null. Log with action MILESTONE_BLOCKED.
       - When status is DEFERRED: set deferredReason from reason, set deferredRevisitDate from revisitDate, clear blockedReason, set completedAt to null. Log with action MILESTONE_DEFERRED.
       - For existing statuses (NOT_STARTED, IN_PROGRESS, SKIPPED): clear blockedReason, deferredReason, deferredRevisitDate on transition away from BLOCKED/DEFERRED.

    4. After the activity log within the same $transaction, add auto-completion check (per D-12):
       - Query all milestones for the same assessmentRecommendationId
       - Define terminal statuses: COMPLETED, SKIPPED, DEFERRED
       - If ALL milestones are in terminal statuses AND none are BLOCKED:
         - Transition the parent AssessmentRecommendation to COMPLETED (set status, completedAt)
         - Log SolutionActivity with action "auto_completed" and detail { reason: "all_milestones_terminal" }
       - If ANY milestone is BLOCKED: do not auto-complete (recommendation stays IN_PROGRESS)
       - Extract this logic into a private async function checkAutoCompletion(tx, recommendationId, actorId)

    5. Create or extend solution-lifecycle.test.ts with unit tests. Since this file uses prisma.$transaction, mock prisma using vi.mock("@/lib/db"). Test cases:
       - updateMilestoneStatus with status BLOCKED sets blockedReason and logs MILESTONE_BLOCKED activity
       - updateMilestoneStatus with status DEFERRED sets deferredReason and deferredRevisitDate
       - Auto-completion triggers when all milestones are COMPLETED/SKIPPED/DEFERRED
       - Auto-completion does NOT trigger when any milestone is BLOCKED
       - Auto-completion does NOT trigger when any milestone is NOT_STARTED or IN_PROGRESS
  </action>
