Create `src/lib/analytics/score-delta.ts` (follow queries.ts pattern with getTrendDirection):
- Import PillarDelta from reassessment-types
- `computePillarDeltas(previousScores: PillarScoreRow[], currentScores: PillarScoreRow[], completedRecommendations: { pillar: string; name: string }[]): PillarDelta[]` -- Pure function (no DB access). For each current score, find matching previous score by pillar. Compute delta = current - previous, round to 2 decimal places. Direction: delta > 0.01 = "improved", delta < -0.01 = "regressed", else "unchanged". Attribution: filter completedRecommendations by pillar. If no completed recommendations for the pillar, attribution = ["No new planning activity"] (per D-06). Handle pillars that exist in current but not previous (new pillar, direction = "improved", previousScore = 0).
- `PillarScoreRow = { pillar: string; score: number; riskLevel: string }` -- Minimal type for the pure function input
- `getScoreDeltasForAssessment(assessmentId: string, previousAssessmentId: string): Promise<PillarDelta[]>` -- Server-side function (import "server-only" guarded). Loads PillarScore rows for both assessments. Loads completed AssessmentRecommendations between the two assessments (status COMPLETED, created after previous assessment's completedAt). Maps through ServiceRecommendation.category to pillar for attribution (per Pitfall 3). Calls computePillarDeltas.
