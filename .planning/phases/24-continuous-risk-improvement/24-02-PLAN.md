Create `src/lib/cadence/cadence-types.ts` (follow src/lib/analytics/types.ts pattern):
- `CadenceStatus = "on_track" | "due_soon" | "overdue" | "system_recommended"` -- matches UI-SPEC cadence states
- `CadenceInfo = { id: string; clientId: string; frequency: CadenceFrequency; nextDueDate: Date; status: CadenceStatus; daysUntilDue: number; isOverridden: boolean; systemRecommended: boolean; systemRecommendationReason: string | null; lastAssessmentId: string | null }`
- `DUE_SOON_THRESHOLD_DAYS = 14` -- per UI-SPEC "Due soon (<=14 days)"
- `CADENCE_FREQUENCY_DAYS: Record<CadenceFrequency, number> = { QUARTERLY: 90, SEMI_ANNUAL: 180, ANNUAL: 365 }` -- per RESEARCH Pattern 4

Create `src/lib/cadence/review-cadence.ts` (import "server-only"):
- `computeNextDueDate(completedAt: Date, frequency: CadenceFrequency): Date` -- Uses date-fns addDays with CADENCE_FREQUENCY_DAYS lookup. Pure function.
- `getCadenceStatus(nextDueDate: Date, systemRecommended: boolean): { status: CadenceStatus; daysUntilDue: number }` -- Uses date-fns differenceInDays. If systemRecommended returns "system_recommended". If daysUntilDue <= 0 returns "overdue". If daysUntilDue <= DUE_SOON_THRESHOLD_DAYS returns "due_soon". Otherwise "on_track". Pure function.
- `getCadenceForClient(clientId: string, advisorProfileId: string): Promise<CadenceInfo | null>` -- Loads ReviewCadence by unique (clientId, advisorProfileId), computes status, returns CadenceInfo.
- `createOrUpdateCadence(input: { clientId: string; advisorProfileId: string; frequency: CadenceFrequency; nextDueDate: Date; isOverridden: boolean }): Promise<CadenceInfo>` -- Upserts ReviewCadence by unique constraint. Per D-08, isOverridden=true when advisor sets it.
- `getOverdueCadences(): Promise<ReviewCadence[]>` -- Finds all ReviewCadence where nextDueDate < now. Used by cron. Filter out rows where lastReminderSentAt is within 7 days (per Pitfall 5 dedup).
- `getDueSoonCadences(thresholdDays?: number): Promise<ReviewCadence[]>` -- Finds ReviewCadence where nextDueDate is within thresholdDays (default 14). Also filters by lastReminderSentAt for dedup.
- `initializeCadenceForClient(clientId: string, advisorProfileId: string, assessmentId: string, assessmentCompletedAt: Date): Promise<CadenceInfo>` -- Creates initial cadence after first assessment. Loads enterprise default frequency from AdvisorEnterprise.defaultCadenceFrequency. Sets nextDueDate = computeNextDueDate(completedAt, frequency). Sets lastAssessmentId.

Create `src/lib/cadence/system-triggers.ts` (import "server-only"):
- `SYSTEM_REASSESSMENT_THRESHOLD = 3` -- number of completed recommendations that triggers system recommendation (per D-09)
- `checkSystemReassessmentTriggers(clientId: string, assessmentId: string): Promise<{ shouldRecommend: boolean; reason: string | null }>` -- Counts completed AssessmentRecommendations for the assessment. If count >= threshold AND no reassessment exists for this assessment (no Assessment with previousAssessmentId = assessmentId), returns shouldRecommend=true with reason string like "3 completed recommendations since last assessment". Per D-09.

Update `src/lib/engagement/feature-flags.ts`:

If no enterprise, return true (solo advisors default enabled). If enterprise exists, return implementationTrackingEnabled (cadence piggybacks on the same flag -- if tracking is off, cadence is off too).
