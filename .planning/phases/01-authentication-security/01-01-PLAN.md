  <action>
    1. Create `src/lib/auth.config.ts` with Auth.js configuration object (separated for Edge compatibility):
       - Use `CredentialsProvider` with email and password fields
       - In `authorize`: find user by email via Prisma, verify password with `argon2.verify()`, return user object or null
       - Configure pages: { signIn: "/signin", error: "/signin" }

    2. Create `src/lib/auth.ts` that imports config from auth.config.ts and creates the Auth.js instance:
       - Use `PrismaAdapter(prisma)` for database persistence
       - Set session strategy to "database" for server-side sessions with instant revocation
       - Export `{ handlers, signIn, signOut, auth }` from `NextAuth()`
       - In session callback: attach user.id and mfaEnabled to session object
       - IMPORTANT: Auth.js v5 with database strategy and CredentialsProvider requires manual session creation in the authorize callback because the Prisma adapter doesn't auto-create sessions for credentials. Handle this by using the `jwt` strategy initially BUT storing critical data server-side, OR by manually creating a session in the signIn callback. Research the Auth.js v5 docs pattern for credentials + database sessions. The cleanest approach: use `jwt` strategy with short expiry (15 min) and encode minimal data (userId only), then look up user data server-side in the session callback. This avoids the credentials+database session limitation.

    3. Create `src/app/api/auth/[...nextauth]/route.ts`:
       ```typescript
       import { handlers } from "@/lib/auth"
       export const { GET, POST } = handlers
       ```

    4. Create `src/app/api/auth/register/route.ts` as POST endpoint:
       - Validate input with Zod schema: email (valid email format), password (min 8 chars, must contain uppercase, lowercase, number, special character)
       - Check if user already exists (return 409 if so, with generic message "Unable to create account" to avoid email enumeration)
       - Hash password with Argon2id: `argon2.hash(password, { type: argon2.argon2id, memoryCost: 65536, timeCost: 3, parallelism: 1 })`
       - Create user via `prisma.user.create()`
       - Return 201 with `{ id, email }` (never return password hash)
       - Wrap in try/catch, return 400 for validation errors, 500 for unexpected errors

    5. Create `src/app/(auth)/layout.tsx` as a centered layout for auth pages (Tailwind: flex items-center justify-center min-h-screen, max-w-md card-style container).

    6. Create `src/app/(auth)/signup/page.tsx`:
       - Client component with form: email input, password input, confirm password input, submit button
       - Client-side validation: passwords match, minimum 8 chars
       - On submit: POST to /api/auth/register, then auto-sign in via `signIn("credentials", { email, password, redirectTo: "/" })`
       - Show error messages inline (zod validation errors from API, "Unable to create account" for conflicts)
       - Link to "Already have an account? Sign in"

    7. Create `src/app/(auth)/signin/page.tsx`:
       - Client component with form: email input, password input, submit button
       - On submit: call `signIn("credentials", { email, password, redirectTo: "/" })`
       - Handle errors: display "Invalid email or password" (generic, no enumeration)
       - Link to "Don't have an account? Sign up" and "Forgot password?"
  </action>

    - POST to /api/auth/register with weak password returns 400
    - Sign in with valid credentials succeeds and creates session
    - Sign in with invalid credentials returns error
    - `/signin` and `/signup` pages render without errors
