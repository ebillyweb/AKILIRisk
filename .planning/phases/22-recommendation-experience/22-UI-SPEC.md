## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline chip gaps |
| sm | 8px | Tight element spacing, badge row gaps |
| md | 16px | Default card padding, form field gaps |
| lg | 24px | Section padding, card header spacing |
| xl | 32px | Inter-section spacing, layout column gaps |
| 2xl | 48px | Major section breaks on client Strategic Action Plan |
| 3xl | 64px | Page-level vertical rhythm on full-page views |

Exceptions:
- Touch targets for Accept/Defer/Decline action buttons: minimum 44px height (use `min-h-11` / `min-h-9 px-4`)
- Inline layer attribution chips: 20px height (`h-5 text-[10px]`) — established pattern from existing `Badge` usage

## Layout Patterns

Three distinct layout contexts in this phase. Each follows the established `hero-surface` + `page-shell` pattern.

### Advisor Guidance Review Page (`/advisor/clients/[id]/guidance`)

- Full-width `page-shell` layout
- `AdvisorPageHeader` with kicker "Client Guidance", title "[Client Name]", subtitle "Holistic guidance package across all completed assessments"
- Top: `hero-surface` Executive Summary strip — readiness score badge, top-3 priority list, last-assessed date
- Below: tabbed nav (`Tabs`) with tabs: Overview | Recommended Actions | Evidence | Deferred | Implementation Plan
- Each recommendation card: `Card` with `CardHeader` (insight title + priority badge + layer attribution row) and `CardContent` (evidence accordion + why it matters + advisor controls row)
- Advisor controls row: `Button` variant="default" "Include" + `Button` variant="outline" "Defer" + `DropdownMenu` with "Already Addressed", "Hide from Client", "Adjust Priority"
- Sticky action bar at bottom of page (fixed position): bulk action count + "Include Selected" + "Save All"

### Enterprise Guidance Customization Page (`/advisor/enterprise/guidance`)

- `page-shell` with two-column layout: left 40% platform definition (read-only, `bg-muted/40`), right 60% enterprise overlay (editable)
- Platform definition column: immutable fields labeled with `Badge variant="outline"` reading "Platform — Protected"
- Enterprise overlay column: form fields for configurable and additions fields, each labeled "Enterprise Overlay"
- Visual separator between columns: `Separator` orientation="vertical"
- Preview toggle button: `Button` variant="outline" "Preview Composed Result" — opens `Sheet` from right with the fully composed view

### Client Strategic Action Plan Page (`/client/action-plan` or embedded in client dashboard)

- Full-width `page-shell`, no sidebar
- Hero: Executive Summary card with `hero-surface` class — overall readiness score (large display number), top priorities list (3 items max), recently completed count
- Sections rendered as vertical stack with `2xl` (48px) spacing between:
  1. **Immediate Priorities (0-90 days)** — `Card` list, each with status badge + progress indicator
  2. **Strategic Initiatives (3-12 months)** — `Collapsible` card per initiative
  3. **Ongoing Practices** — compact `Card` with recurring cadence label
  4. **Progress Dashboard** — `Progress` bars per active recommendation + completion percentage
- Each action card: title, why-it-matters (body text, 2-line clamp with expand), status selector (client-managed `Select`), owner role badge, implementation guidance (collapsible via `Collapsible`)

## Interaction Contracts

### Advisor Controls (D-05)

- **Include in Action Plan:** `Button` variant="default" (primary accent) — immediate optimistic UI update, badge transitions to "In Action Plan". No confirmation dialog required.
- **Defer:** `Button` variant="outline" — opens `Dialog` requiring: reason (`Select` from predefined list), optional revisit date (`Input` type="date"), optional trigger event (`Input` text). "Confirm Defer" button in dialog.
- **Mark Already Addressed:** `DropdownMenu` item — immediate status change, no dialog. Badge transitions to "Completed".
- **Hide from Client:** `DropdownMenu` item — confirmation `Dialog` with destructive confirm button ("Yes, hide from [Client Name]"). Uses `--destructive` color.
- **Add Advisor Notes:** Inline `Textarea` that expands on click, auto-saves on blur (no explicit save button).
- **Adjust Priority:** `DropdownMenu` submenu with "High / Medium / Low" — immediate optimistic update.

### Enterprise Overlay (D-14, D-15)

- All overlay fields save via a single "Save Overlay" `Button` at bottom of right column.
- Unsaved changes indicator: yellow dot on tab label when form is dirty.
- "Preview Composed Result" opens `Sheet` — read-only, no actions within preview.

### Client Action Tracking (D-21)

- Task status: `Select` dropdown per action card — client-managed. Transitions are immediate with optimistic UI.
- Validation status: advisor-only badge (not a client control). Displayed as read-only `Badge`.
- "Mark Complete" shortcut `Button` variant="outline" on each action card — sets Task Status = Completed, triggers advisor validation request if required by platform/enterprise config.

### Bulk Actions (Advisor Guidance Review)

- Checkbox on each recommendation card (top-right corner, `Checkbox` component).
- Selecting any card reveals sticky bottom bar with count ("3 selected") and bulk CTAs.
- Bulk "Include All Selected": no dialog. Bulk "Defer All Selected": single dialog, applies same reason/date to all selected.

## Accessibility Requirements

- All interactive advisor controls (Include, Defer, dropdowns) must be keyboard-reachable and announce state changes via `aria-live="polite"` on the status badge region.
- Layer attribution badges must include `aria-label` clarifying source: e.g. `aria-label="Source: Platform"`.
- Progress bars use `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`.
- Dialog components use existing shadcn `Dialog` which handles focus trap and `aria-modal="true"` automatically.
- Color is never the sole differentiator for lifecycle status — always pair badge color with text label.
